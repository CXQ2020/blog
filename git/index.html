# git教程

# 基本操作

  1. git init 初始化一个git仓库，可以在当前目录运行，也可以指定目录运行

  2. git add 文件添加到仓库，

    git add . 添加全部，
    git add filename 添加指定文件
    git add *.py 使用通配符进行文件过滤

  3. git commit 提交打仓库

    git commit -m '本次提交注释'

  4. git clone 从git仓库中拷贝项目 git

    git clone <repo> <directory>
    参数说明

    • repo: Git 仓库
    • directory:本地仓库
      例如 git clone git://github.com/schacon/grit.git mygrit

  5. git status 查看当前项目的状态 参数 -s 使用简短输出

  6. git diff 查看执行 git status结果的详细说明

    • 尚未缓存的改动: git diff
    • 查看已缓存的改动: git diff --cached
    • 查看已缓存的与未缓存的所有改动 git diff HEAD

  7. git reset HEAD 命令用于取消已缓存的内容

    git reset HEAD hello.py 取消hello.py的缓存

  8. 要从 Git 中移除某个文件，就必须要从已跟踪文件清单中移除，然后提交。可以用以下命令完成此项工作

    git rm <file>

    如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f

    git rm -f <file>

    如果把文件从暂存区域移除，但仍然希望保留在当前工作目录中，换句话说，仅是从跟踪清单中删除，使用 --cached 选项即可

    git rm --cached <file>

    如我们删除 hello.php文件：

    $ git rm hello.php 
rm 'hello.php'
$ ls
README

    不从工作区中删除文件：

    $ git rm --cached README 
rm 'README'
$ ls
README

    可以递归删除，即如果后面跟的是一个目录做为参数，则会递归删除整个目录中的所有子目录和文件：

    git rm –r *

    进入某个目录中，执行此语句，会删除该目录下的所有文件和子目录。

  9. git tag -a v1.0 -m '标签备注' 给仓库代码打上标签

# 分支管理

  1. 查看分支

    git branch 查看所有分支 标记* 的为当前分支

  2. 创建分支

    git branch branchname 创建分支

  3. 切换分支

    git checkout branchname 切换分支
    git checkout -b branchname 创建并切换到新创建的分支

  4. 合并分支

    git merge
    先切换到合并到的目标分支，然后使用命令git merge branchname 合并到目标分支上

  5. 删除分支

    git branch -d branchname 删除指定名称的分支

# git查看提交历史

Git log 命令列出历史提交记录

  1. git log 列出历史提交记录
  2. git log --oneline 简洁的历史提交记录
  3. git log --graph 开启拓扑图的历史提交记录
  4. git log --author 指定用户的提交记录
  5. git log --since 和 --before， --until 和 --after 指定日期
